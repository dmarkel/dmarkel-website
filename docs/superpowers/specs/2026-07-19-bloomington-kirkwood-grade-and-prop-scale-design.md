# Bloomington Kirkwood Grade and Prop Scale Correction Design

## Goal

Correct two independent depth problems in the Bloomington scene: Kirkwood architecture and street activity sit above the visible sidewalk grade, while modular curbside props render substantially smaller than the avatar. Preserve the approved Sample Gates position, scene pacing, sidewalk, avatar, physics, and Nick's endpoint.

## Root Cause

The second environment source combines Sample Gates and Kirkwood in one transparent panel, but their baked baselines differ. Sample Gates reach source y=736 and are correctly aligned by the current panel offset of -70. Kirkwood reaches only source y=719, so the same offset places its baseline at y=649, 16 source pixels above the foreground sidewalk edge at y=665.

The avatar and foreground props also use different screen-size contracts. The avatar uses a responsive player scale whose minimum is 1.5, while prop dimensions and base anchors use the scene-art scale, which is approximately 1.0 in the reviewed portrait and landscape layouts. Moving props to the nearer curb row exposed this pre-existing mismatch.

## Chosen Approach

### Preserve Sample Gates and lower only Kirkwood

Create a deterministic, versioned replacement for the second environment panel. Use the safe architectural boundary at the first Kirkwood building edge near source x=815:

- shift the Sample Gates and transition region left of the boundary upward 16 pixels inside the replacement asset;
- leave the Kirkwood region at and right of the boundary at its existing source position;
- change the runtime offset for the whole replacement panel from -70 to -54.

The combined transforms preserve the Sample Gates' effective screen position: -16 pixels inside the asset plus a -54 runtime offset equals the current -70 placement. Kirkwood receives only the -54 runtime offset, moving it down 16 pixels so its source y=719 baseline meets the sidewalk edge at y=665.

The boundary must remain at the existing building edge so no person, vehicle, gate pylon, arch, tree trunk, or storefront is split. The source panel remains untouched; the replacement is generated by a checked-in deterministic build tool.

### Match curbside props to the avatar scale

Extend the shared prop transform with an optional visual-size scale that is independent of world-position scale. World x coordinates and ground-plane y coordinates continue to use the scene scale. Width, height, and `baseY` use the visual-size scale.

Bloomington passes the current responsive avatar scale as the visual-size scale. Houston and all callers that omit the new argument retain their existing behavior.

Larger prop images expand around their existing horizontal centers, not from the left edge. Every prop's base remains anchored to the y=765 curb plane. This keeps the bench, campus lamp, planter, newspaper box, parking meter, and bike rack in their approved world locations while making their proportions consistent with the avatar.

## Alternatives Considered

### Lower the complete second environment panel

Rejected because it would move the approved Sample Gates and would either leave Kirkwood inaccurate or recreate the buried pedestrian and vehicle problem.

### Lower Kirkwood by approximately 50 pixels

Rejected without a full street-activity re-layer. A 50-pixel drop would put the Kirkwood baseline at y=699, 34 pixels behind the opaque sidewalk. The measured 16-pixel correction reaches the exact visible grade without clipping.

### Apply arbitrary per-prop scale multipliers

Rejected because it would create inconsistent proportions across viewports. The avatar's existing responsive scale is the correct shared visual reference.

## Rendering Contract

- Sample Gates effective offset: -70 source pixels, unchanged.
- Kirkwood effective offset: -54 source pixels.
- Kirkwood baseline: source y=719 plus offset -54 equals sidewalk edge y=665.
- Avatar floor: source y=735, unchanged.
- Curbside prop baseline: source y=765, unchanged.
- Prop world position uses scene scale.
- Prop visual dimensions and base anchor use avatar scale in Bloomington.
- Prop horizontal center remains unchanged when visual scale differs from scene scale.
- Curbside props continue to render after the avatar and remain non-collidable.
- Nick's right edge and world endpoint remain source x=3812.

## Error Handling

The asset build must reject an unexpected source size, a non-RGBA source, or a boundary that falls outside the image. The prop transform must preserve its existing default when no visual-size scale is supplied and reject non-positive or non-finite scale values.

## Verification

Automated tests must prove:

- the new environment asset is exactly 1906x825 RGBA;
- the Sample Gates' effective opaque bounds are unchanged after runtime offset;
- the Kirkwood baseline moves from effective y=649 to y=665;
- the split boundary does not cut a large connected landmark;
- no visible magenta contamination or unintended partial alpha is introduced;
- the second environment runtime offset is exactly -54;
- a prop's ground anchor remains fixed when its visual-size scale changes;
- a prop's horizontal center remains fixed when its visual-size scale changes;
- omitting the visual-size scale preserves the existing Houston transform;
- all six Bloomington curb props use the avatar scale;
- existing movement, jump, viewport, foreground, asset-integrity, and endpoint tests continue passing.

Browser review must inspect portrait and landscape layouts at Kelley, Sample Gates, the transition boundary, Kirkwood pedestrians and cars, all six curb props, and Nick's. Sample Gates must not move. Kirkwood feet, wheels, and storefront bases must meet the sidewalk grade. Props must read as naturally scaled near-sidewalk objects, remain grounded, and occlude the avatar only where their visible pixels overlap.

## Reusable Scene-Pack Protocol Addition

Future panels must record landmark and activity baselines independently before combining them. If one source contains regions with different baked grades, align them during deterministic asset assembly rather than applying one compensating runtime offset to the entire panel. Foreground props intended to share depth with an independently scaled avatar must separate world-position scale from visual-size scale and preserve both center and base anchors.
